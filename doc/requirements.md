# 《时间遗失处》需求文档

## 1. Overview

《时间遗失处》是一款单局约 4–6 分钟的竖屏人物判断游戏：玩家查看一位误入当代的陌生人，点开最多 3 条线索，然后决定“留下”或“送回”。

- 核心体验：先看人，再查细节，最后做一个直观但有代价的决定。
- 玩家不需要理解机构、时间物理或评分术语才能操作。
- 世界观通过人物对白、线索和裁定后果逐层显露，不在主界面一次性罗列。
- 每局 7 位来客，每位 24 秒；打开线索时暂停计时。

### 屏幕状态

1. 开始页：海报、游戏名、一句玩法和“开始”。
2. 判断页：人物图、姓名、对白、3 个线索热点、倒计时和“留下 / 送回”。
3. 线索页：一次只展示 1 条线索，关闭后回到同一人物。
4. 揭晓第一层：人物图上明确显示“判对了 / 判错了”，等待玩家点击“看看为什么”。
5. 揭晓第二层：展示决定性线索和一句后果，玩家点击“下一位”。
6. 暂停页：继续、声音和重新开始。
7. 结算页：正确数、简短结局和“再来一局”。

## 2. Visual Design

- 保留现有 7 张人物肖像与海报，不重新生成角色素材。
- 视觉方向由“复杂档案终端”改为“深夜人物窗口”：一屏只突出一个人和一个问题。
- 主画面至少 58% 高度用于人物；信息层以对白条、线索热点和底部双按钮叠加，不再堆叠资源卡、仪器卡和说明标题。
- 主界面不出现“因果稳定、记忆连续、因果回声、协议”等专有名词。
- 线索入口只显示编号和放大镜；自然语言解释仅在玩家点开后出现。
- 具体色彩、排版、图标、层级与动效写入 `doc/visual.md`。

## 3. Game Mechanics

- 每局固定 7 位来客，顺序保持叙事依赖。
- 每位来客初始 24 秒；人物加载不消耗时间，线索抽屉和暂停状态冻结计时。
- 3 条线索均可免费查看；已查看线索用实心标记表示。
- 玩家可在任意时刻选择“留下”或“送回”，提交后不可撤销。
- 每案存在 1 个由关键线索支持的正确判断；第 4 案会读取第 2 案选择并动态改变答案。
- 单案得分：基础 100；看完 3 条线索 +25；剩余至少 8 秒 +20；判断正确 +35。单案最高 180。
- 内部仍记录两项城市后果值，初始均为 60，范围 0–100；它们不占用判断页，只用于后果文案、提前结局和最终城市结局。
- 提交后先停留在人物揭晓层；只有玩家主动点击才展开文字解释，再主动进入下一位。
- 图片加载超过 1.2 秒或失败时必须解除遮罩，以人物剪影和完整文字继续游戏。

## 4. Controls

- 点击人物图上的 3 个线索热点打开对应线索；热点至少 44×44 CSS px，使用 `onClick`。
- 点击遮罩、关闭按钮或“看完了”关闭线索；倒计时恢复。
- “留下”“送回”和“开始”使用 `onPointerDown`，按下立即锁定，防止重复提交。
- 揭晓阶段“看看为什么”“下一位”使用 `onClick`。
- 暂停、声音、语言切换使用 `onClick`。
- 桌面鼠标与移动端触控使用相同规则；页面进入后台自动暂停。
- 支持 320×568 至 430×932，并避让上下安全区；中文与英文均不得横向溢出。

## 5. Win / Lose Conditions

- 完成 7 位来客即完成值班；结算首先显示“正确数 / 7”。
- 任一内部城市后果值降至 0 时，在当前解释阅读完后提前结束，并用自然语言说明结局，不显示抽象失败代码。
- 结算根据两项后果值生成一句结局：城市更稳定、更多人被保住、两者平衡或两者均受损。
- 结算显示正确数、总分、历史最佳和一个“再来一局”按钮。

## 6. Sound Effects

- 开始：220→330 Hz 正弦上行，180 毫秒，音量 0.08。
- 打开线索：540→680 Hz 正弦短音，75 毫秒，音量 0.07。
- 关闭线索：480→350 Hz 正弦下行，70 毫秒，音量 0.045。
- 提交选择：85 Hz 正弦叠加 45 毫秒噪声，90 毫秒，音量 0.13。
- 判对：392→523→659 Hz 正弦短音阶，240 毫秒，音量 0.09；可用时震动 18/35/24 毫秒。
- 判错：260→170 Hz 锯齿下行，180 毫秒，音量 0.08；可用时震动 55 毫秒。
- 下一位：420→520 Hz 正弦短音，90 毫秒，音量 0.06。
- 静音写入 `localStorage`；音频失败不得阻塞游戏。
