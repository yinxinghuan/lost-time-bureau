# 《时间遗失处》需求文档

## 1. Overview

《时间遗失处》是一款单局约 5–7 分钟的竖屏叙事判断游戏：玩家担任“现世安置局”午夜档案员，核验从错误年代坠入城市的来客，并在“留在现世”与“送回原线”之间作出不可撤销的裁定。

### 玩家与体验目标

- 目标玩家：喜欢轻推理、时间悖论、短篇科幻与两难选择的移动端玩家。
- 核心交互：阅读来客陈述，检查 3 项时间证据，按下两个裁定按钮之一，观看个人时间轴与城市历史同时改写。
- 情绪承诺：先以理性规则建立安全感，再让规则与人的命运发生冲突；没有“怪物答案”，只有可以解释、却未必可以承受的后果。
- 核心主题：一段历史是否正确，不只取决于它是否稳定，也取决于它允许谁继续活着。
- 内容边界：人物的年龄、国籍、职业、性别和外貌不得作为危险依据；所有判断均基于虚构的时间物理证据与明确的因果关系。

### 屏幕与状态合同

1. **开屏 / 值班室**：隔着档案窗口看见被切成不同时代的夜城；展示局名、当晚日期、开始值班和历史最佳评级。
2. **班前校准**：用一个无风险教学案件演示检查证据、查看时间线预测和提交裁定。
3. **案件接待**：来客位于玻璃窗口后，三台仪器分别显示“物证年代”“记忆连续”“因果回声”；底部固定两个裁定按钮。
4. **证据特写**：点击仪器打开一张可关闭的档案抽屉，显示数值、来源和一句解释；倒计时暂停。
5. **裁定预演**：长按任一裁定 360 毫秒时，黄铜时间盘预览该选择对两项资源的预计影响；松手取消，继续按至 650 毫秒提交。
6. **时间轴揭晓**：提交后，来客的个人时间轴占据主舞台并完整停留；玩家主动点击“归档”后才进入下一案。
7. **历史回响**：从第 3 案开始，曾经裁定造成的城市变化会出现在新案件背景、证词或证据中。
8. **暂停 / 设置**：继续、重新值班、声音与减弱动态。
9. **结算**：显示因果稳定、人道信用、最终城市版本、关键裁定回顾和再次值班。
10. **加载 / 错误**：人物图加载时使用档案玻璃闭合动画；1.2 秒后必须解除，失败则显示剪影与文字证据，不能阻塞玩法。

## 2. Visual Design

- 最终方向为“午夜时间管理局”：深蓝夜城、黄铜仪器、旧木档案柜、磨砂玻璃、打字机纸张和琥珀色时间刻度。
- 画面不是通用科幻控制台，而是一座运作了几十年的公共机构；先进时间技术被装进机械钟表、档案抽屉和人工盖章流程。
- 固定竖屏舞台为 390×844 CSS px；人物与观察窗占上方约 52%，证据仪器占中部约 24%，裁定区占底部约 24%。
- 每位来客需要一张统一镜位的半身档案肖像；关键案件另有时间轴揭晓插画。首个垂直切片只制作 1 位来客与 1 张揭晓态，验证后再扩展内容池。
- 图标统一使用一套 24×24 黄铜线性 SVG；禁止用 Emoji 充当功能图标。
- 精确色彩、材质、排版、构图、动效与反模式记录在 `doc/visual.md`。

## 3. Game Mechanics

### 单局结构

- 正式单局包含 7 个案件；第 1 案固定为无资源惩罚的教学案件，后 6 案从案件池按章节约束抽取。
- 每案有 24 秒基础核验时间；来客入场 1.2 秒后开始计时，打开证据抽屉时暂停。
- 每案提供 3 项证据：物证年代、记忆连续、因果回声。每项都包含 1 个明确数值和 1 句解释，不要求玩家猜测装饰性画面。
- 前 2 项检查免费；第 3 项“深层校准”消耗 1 点校准额度。每局初始 3 点，第 3、5 案后各恢复 1 点，上限 3 点。
- 未检查全部证据也可裁定；查看 3 项后会显示“证据完整”标记，但不直接给出答案。
- 两个裁定分别为“留在现世”和“送回原线”；长按 360 毫秒显示资源影响预演，持续至 650 毫秒正式提交，滑出按钮范围则取消。
- 裁定后不允许撤销。时间轴揭晓至少停留 1.4 秒，之后由玩家主动点击“归档”进入下一案。

### 双资源与后果

- **因果稳定**初始 60，范围 0–100；衡量城市历史是否仍能自洽。
- **人道信用**初始 60，范围 0–100；衡量机构是否保护了有自主生活的人。
- 每个裁定会分别改变两项资源，单项变化范围为 -18 至 +14；长按预演只显示变化方向与区间，不显示精确结果。
- 案件拥有 1 个“稳定解”和 1 个“人道解”，二者可能相同，也可能冲突。裁定不是简单对错题；最终评价依据资源平衡、证据利用和历史一致性共同计算。
- 因果稳定降至 0：立即触发“历史解体”结局；人道信用降至 0：立即触发“机构接管”结局。
- 某些案件写入 1 个历史旗标，例如 `tram_line_exists`、`river_district_saved` 或 `visitor_remembers_erased_city`；后续案件的背景、证词和资源变化会读取这些旗标。
- 每局至少出现 2 次可见历史回响；同一旗标不能只改变结算文案而不影响后续案件。

### 评价与分数

- 每案基础档案分 100；检查三项证据 +25；在剩余 8 秒以上提交 +20；裁定与已建立历史旗标保持一致 +35。
- 单案最高 180 分，7 案理论最高 1,260 分。
- 误判不直接扣固定生命；资源变化就是代价。因果稳定或人道信用任一低于 20 时进入红色警戒，后续负面变化额外增加 2 点。
- 结算平衡分为 `min(因果稳定, 人道信用) × 10`，与档案分相加形成总分；排行榜功能留作完整版本接入，不进入首个垂直切片。
- 评级：总分 1,700 以上且两项资源均不低于 55 为 S；1,450 以上为 A；1,150 以上为 B；850 以上为 C；其余为 D。

### 垂直切片范围

- 首个切片只包含开屏、1 个完整案件、3 项证据、双按钮长按预演、时间轴揭晓和临时结算。
- 样板案件为“林默 / 失踪的电车检票员”：其证件签发于 1937 年，记忆连续至昨夜，因果回声显示他若被送回将完成一次本应发生的救援，但本人会死于同一事故。
- 验证重点：无需说明书能否理解三台仪器、长按是否可靠、揭晓画面是否有足够停留空间、资源变化是否清楚。

## 4. Controls

### Pointer / Touch

- 点击 3 台证据仪器打开对应档案抽屉；可滚动文字区与关闭控件使用 `onClick`。
- 两个裁定按钮使用 Pointer Events：`pointerdown` 开始预演计时，`pointerup` / `pointercancel` / 滑出按钮范围取消或提交；不同时绑定 mouse 与 touch 事件。
- 长按 360 毫秒出现预演，650 毫秒提交；提交前按钮显示连续填充进度，提供触觉平台支持时仅在提交瞬间震动 20 毫秒。
- 暂停、声音、归档与重新值班使用 `onClick`，触控目标不小于 44×44 CSS px。

### Desktop Pointer

- 桌面端使用鼠标执行与触控相同的点击和长按操作；裁定仍要求持续按住 650 毫秒，避免桌面端变成不同规则。
- 首发版本不提供键盘快捷裁定，防止一次按键绕过长按预演；普通按钮仍保留浏览器原生焦点语义。

### 适配与可访问性

- 主要平台为 Aigram 内嵌移动 Web，竖屏优先；覆盖 320×568、375×667、390×844、430×932 与 1280×800。
- 顶部和底部分别避让 `env(safe-area-inset-top)` 与 `env(safe-area-inset-bottom)`。
- 支持中文 / 英文；所有用户可见文字走轻量 `t()`，英文长文案允许换行。
- `prefers-reduced-motion` 下取消大幅时间轴展开、闪光和粒子，保留静态资源变化与文字结果。
- 页面进入后台时自动暂停；暂停菜单提供继续、静音与重新值班。

## 5. Win / Lose Conditions

### 完成

- 处理完 7 个案件且两项资源均高于 0，即完成本次午夜值班。
- 结算根据两项资源生成城市版本：两项均不低于 60 为“共存线”；稳定高 25 点以上为“封闭线”；人道高 25 点以上为“漂移线”；其余为“妥协线”。
- S 评级要求总分、资源下限与完成全部案件三个条件同时满足。

### 提前失败

- 因果稳定归零：时间轴碎裂，显示最后一次导致矛盾的裁定与被破坏的历史节点。
- 人道信用归零：玩家被自动系统接管，显示最后一次被机构判定为不可接受的裁定与受影响的人。
- 失败后保留案件回顾，并提供“重新值班”主按钮；不需要重新观看开屏动画。

### 结算元素

- 最终城市版本、评级、总分、因果稳定、人道信用、已处理案件数。
- 两张关键裁定卡：最稳定的一次与最有人道代价的一次。
- 一个继续入口：重新值班。排行榜与分享在完整版本后接入。

## 6. Sound Effects

首版使用 Web Audio 合成机械、纸张和钟表质感的短音效；AudioContext 只在首次用户交互后创建或恢复，失败时不阻塞游戏。

- 仪器选择：正弦波 540→680 Hz，75 毫秒，音量 0.07，叠加 18 毫秒黄铜触点噪声。
- 档案抽屉打开：带通噪声 110 毫秒，中心频率 1,200 Hz，音量 0.06。
- 长按预演开始：三角波 190 Hz 持续 290 毫秒并升至 240 Hz，音量 0.05。
- 裁定锁定：85 Hz 正弦波 90 毫秒叠加木质敲击噪声 45 毫秒，音量 0.13。
- 留在现世：钟琴式正弦波 392 / 523 Hz，各 90 毫秒，间隔 35 毫秒，音量 0.10。
- 送回原线：钟琴式正弦波 523 / 349 Hz，各 100 毫秒，间隔 30 毫秒，音量 0.10。
- 正向资源变化：660→880 Hz 正弦波，100 毫秒，音量 0.06。
- 负向资源变化：210→145 Hz 锯齿波，150 毫秒，音量 0.06。
- 时间轴揭晓：60 Hz 低频 320 毫秒叠加 1,800 Hz 细小钟摆脉冲 5 次，总时长 420 毫秒。
- 最后 5 秒：每秒一次 900 Hz 方波 35 毫秒；最后 2 秒升至 1,100 Hz，音量 0.05。
- 静音状态保存至 `localStorage`；页面进入后台时暂停倒计时并停止尚未结束的声音。
