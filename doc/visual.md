# 《The Lost Time Office》视觉圣经

> 已选并已迁移方向：Modern Civic System。旧式官僚档案方向全部作废；独立 HTML 垂直切片已完成评审，正式游戏现使用本文件描述的视觉系统。

## 1. Visual thesis

- 游戏与用户：面向英语用户为主的竖屏人物判断游戏，单局 4–6 分钟。
- 情绪承诺：玩家面对的是一个可信、冷静但涉及真实人命的公共决策系统。
- 一句话视觉命题：像一个设计严谨的当代公共机构产品，而不是旧文件、科幻仪表盘或阅读应用。
- 标志性瞬间：人物肖像被硬黑网格框住，左上红色案件编号与右下公共蓝裁定按钮形成稳定的视觉轴；判定后整块编号栏扩展成明确的 `CORRECT / INCORRECT` 反馈。
- 必须具备：清晰、现代、果断。
- 必须避开：复古档案、模板化 Dashboard、赛博朋克。

## 2. Composition and camera

- 竖屏响应式 DOM；设计基准 390×844，覆盖 320×568 至 430×932，并支持仅 390 px 高的桌面横向窗口。
- 根容器始终等于 `100dvh`；主状态使用内部 CSS Grid 分配高度，禁止固定截图高度、整页缩放或让底部操作掉出视口。
- 人物肖像占主判定页约 62% 高度，保持胸像正面视角，眼睛位于上三分线。
- 顶部 72 px 为稳定 HUD：56 px 案件编号、标题/状态、44 px 菜单或地图入口。
- 底部信息区按“对白 → 线索进度 → 两个裁定按钮”排列，不与人物脸重叠。
- 展开页使用整屏内容，不叠半透明底层；上方图示、中央事实网格、下方解释与单一返回按钮。
- 注意路径：案件编号 → 人脸 → 线索热点 → 对白 → 裁定按钮。

## 3. Color

- 结构黑 `#101820`：正文、边框、主按钮、外部背景。
- 系统白 `#F1F2EF`：主要界面底色。
- 中性灰 `#C9CDD0`：评审背景、禁用区和次要分隔。
- 次要文字 `#5F666A`：状态与说明。
- 公共蓝 `#2E61D5`：主裁定、地图主路径、当前节点、焦点。
- 警示红 `#F14D32`：案件编号、错误、危险与关键提醒。
- 成功绿 `#16856F`：正确裁定与稳定状态；同时配合图标和文字，不单靠颜色。
- 使用比例：黑/白/灰 85%，蓝 10%，红与绿合计不超过 5%。
- 禁止蓝红渐变、霓虹发光、高饱和大面积背景和半透明玻璃层。

## 4. Typography

- 全界面只使用现代无衬线：`"Helvetica Neue", Arial, "Noto Sans SC", sans-serif`。
- 游戏名 13 px / 800；屏幕标题 24–32 px / 800；正文 16 px / 600；按钮 16 px / 800。
- 工具标签 9–11 px / 800，使用大写与 `0.08–0.12em` 字距。
- 数字与倒计时沿用同一字体，用重量和 tabular numerals 区分，不引入等宽或衬线字体。
- 英文标题不使用全宽字距；长文案允许自然换行。中文正文最低 16 px。

## 5. Shape, material, and lighting

- 主导几何为直角矩形、2 px 结构线和 56 px 编号色块。
- 常规圆角为 0；只有手机外壳或系统级圆形节点可使用圆角。
- 主要按钮使用 2 px 黑边与 3–4 px 错位实体阴影；按下时阴影归零并位移 3 px。
- 不模拟任何真实材质；背景保持平面、干净、无纹理。
- 人物素材保留原有灯光，不在图片上增加泛黄、颗粒、扫描线或滤镜。

## 6. Characters, environments, and assets

- 保留 7 张 1024×1024 人物肖像；使用 `object-fit: cover`，不裁脸。
- 21 张线索插画作为事实图示使用，裁切在 2 px 黑色边框内；不让插画中的装饰决定界面色彩。
- 海报只用于开始页上半部，必须保持英文标题，不出现中文或其他伪文字。
- 人物加载失败显示结构黑底、姓名首字和明确 `IMAGE UNAVAILABLE`，仍可继续游戏。
- 图示失败显示带 SVG 路线图标的空状态与完整 DOM 文案。

## 7. UI and icons

- 使用现有 `LineIcon` 的单线 SVG 体系：24×24，1.8 px 线宽，直线端或极小圆端。
- 功能图标不使用 Emoji；图标按钮必须有可访问名称，触控区至少 44×44 px。
- 主按钮为公共蓝底；次按钮为系统白底黑边；危险/错误只用红色块和错误符号。
- HUD 只保留案件编号、进度/时间和地图或菜单入口。
- 默认：2 px 黑边；按下：下移 3 px；焦点：3 px 蓝色外框；禁用：灰底、斜线图案和文字 `UNAVAILABLE`；加载：保留布局并显示线性进度。
- 成功与失败必须同时显示图标、明确英文文字和一句原因。

## 8. Motion and VFX

- 动效性格：短、硬、机械但不复古。
- `press` 70 ms；页面切换 160 ms；事实面板进入 180 ms；判定冲击 220 ms；地图路线绘制 320 ms；结算数字 360 ms。
- 判定提交：按钮同帧压下；80 ms 后编号栏横向扩展；160 ms 显示结果文字；220 ms 稳定。
- 正确：蓝转绿的单次色块替换与勾形 SVG；错误：红色块与叉形 SVG，人物不抖动。
- 禁止粒子、发光、纸屑、扫描线和持续脉冲。
- `prefers-reduced-motion` 下只保留 100 ms 透明度切换，结果文字立即出现。

## 9. References translated into principles

- 瑞士国际主义：使用严格网格、清楚层级和高信息密度下的可读性；不复制海报式倾斜文字。
- 现代公共交通系统：编号、线路、状态颜色和明确方向；不复制具体城市品牌。
- 当代公共服务网站：可信、可访问、任务导向；不复制普通网页卡片或表单布局。
- 原 C 方案：保留编号红块、公共蓝、硬边框和错位阴影；扩展到所有状态时必须维持同一网格。

## 10. Anti-patterns

- 禁止旧纸、米黄仿古、红色登记线、打孔、手写体、盖章和打字机字体。
- 禁止衬线标题、等宽正文、玻璃拟态、模糊、柔和渐变、霓虹和圆角卡片墙。
- 禁止把每个数据都放进独立卡片；事实只用一组共享网格。
- 禁止专有名词常驻主判断页，禁止用颜色作为唯一结果信息。
- 禁止头像缩小到信息面板的附属地位，人物始终是判定页的第一焦点。

## 11. Vertical-slice acceptance

- 开始页：3 秒内看懂游戏名、人物冲突与唯一开始按钮；默认英文。
- 主判定：第一眼看到人物、3 个线索和两个裁定按钮；HUD 不抢人物。
- 线索：图片、两个关键事实和一句后果在单屏内完成；返回路径明确。
- 高反馈：正确与错误都在 220 ms 内用文字、图标和颜色回答“成功还是失败”。
- 时空图：7 个节点、当前节点、已完成方向与城市走向一眼可读。
- 结算：正确数、城市结局与重新开始按钮在首屏可见。
- 320×568、390×844、430×932 与 844×390 无页面级横纵向溢出；所有可见按钮至少 44×44 px。
- 短屏压缩标题、间距、人物区和说明图；长屏优先把新增空间分配给人物肖像。地图列表与长说明只在内容区内部滚动。
- 垂直切片保留为设计参考；正式 React 游戏已完成同方向迁移。
