# 《时间遗失处》视觉 QA 报告

## Context

- 范围：全局“深夜人物窗口 × 旧式官僚档案”换肤，覆盖开始页、HUD、判断页、线索、状态、时空图、揭晓与结算；保留现有海报、7 张人物肖像和 21 张说明书插画。
- 视觉参考：暖灰旧纸、细红登记格、暗蓝打字墨、手写批注、装订孔与任务手册式图表。
- 对照目标：《午夜验客》的“人物优先、热点查看、分层揭晓”交互。
- 尺寸：390×844、320×568；中文、正常动态。
- 证据：`_qa/ui/*.png` 与 `_qa/ui/report.json`。

## Executive assessment

- Decision：Pass。
- P0 / P1 / P2：0 / 0 / 0。
- 主游戏维持人物优先；档案质感集中在信息层和边框，不用纸纹遮盖人物肖像。
- 点击弹窗完整采用档案页结构，主界面配色同步转为炭黑桌面、暖灰纸、暗蓝墨与暗红登记线。
- 主判断页已移除资源条、三张仪器卡、职业年代字段、操作说明标题与按钮副标题。
- 常驻内容只剩人物、姓名、对白、3 个热点、可点击状态栏和两个判断按钮；线索细节、判断依据与后果均逐层展开。

## Foundation audit

- 功能 Emoji：0；放大镜、对错、暂停、关闭与声音均使用同一组 SVG。
- 触控目标：两种尺寸所有可见按钮均 ≥44×44 px。
- 颜色独立性：判断按钮有文字；结果同时使用图标、巨大文字与颜色。
- 溢出：两尺寸均满足 `body.scrollWidth === clientWidth`。
- 控制台：两次完整 7 人流程均 `errors: []`。
- 状态：开始、判断、状态资讯、暂停、线索图形、线索细节、判对、判错、选择对照、后果时间线和结算均有真实截图。
- 新状态：时空图初始态与完成 3 次裁定后的实时态均在 390×844、320×568 留有截图；地图节点、内部滚动、暂停恢复和返回路径可达。
- 本地化：额外检查英文 320×568 的人物 HUD 与时空图；`CURRENT`、`MAP`、长人物名和 `BACK TO VISITOR` 均无溢出，触控目标保持 ≥44 px。
- 材料层：开始页、判断页、线索抽屉、状态页、时空图、揭晓和结算均保留暖灰纸、登记线和装订边；所有纸纹对比度低于正文，不影响中文或英文阅读。
- 海报：1024×1024 原图与 160×160 列表缩略图均只保留英文标题 `THE LOST TIME OFFICE`；中文副标题已移除，标题位于顶部安全区，人物冲突在缩略图下仍可辨认。

## Iteration evidence

- 初版问题：高屏开始页使用 `object-fit: cover`，海报上方英文标题被横向裁切。
- 修复：海报改为固定上限高度与 `object-fit: contain`，高屏和短屏均保留完整标题和主体。
- 第二轮问题：线索与判定弹窗仍以纯文字为主，用户必须阅读长段落才能理解。
- 修复：三类线索加入日期环、记忆轨迹与去留分叉；线索细节改为可选第二层；判定改为“对错 → 选择对照 → 双节点时间线”三层；顶部状态栏可展开人物资料、城市状态与规则。
- 第三轮问题：程序化日期环、轨迹和分叉仍像占位图，重要后果画面缺少固定展示位置。
- 修复：通过 Aigram transit 制作 21 张统一的无信息文字说明书插画，按“随身物品 / 他说的过去 / 会发生什么”分别展示；读数由 DOM 叠加，`echo` 图在后果层持续展示到玩家主动点击下一位。
- 加载复验：每案 3 张插画在人物出现时后台预载；加载前或失败时原有结构线稿立即可见，图片请求不阻塞倒计时、按钮或裁定。
- 第四轮问题：左上角孤立的 `01/07` 不像入口，也无法快速确认它属于当前人物；整局的世界变化缺少一个可反复查看的总览。
- 修复：状态入口加入 34 px 当前人物头像、“当前来客”、倒计时和进入箭头；旁边新增带短标签的时空图入口。地图以 7 个交错头像节点展示已留下、已送回、当前与未来状态，并实时绘制红/青路线与城市双流向。
- HUD 复验：390 px 下人物入口宽 262 px，320 px 下宽 192 px；时空图入口 52×44 px，暂停 44×44 px，三者均不挤压人物窗口或产生横向溢出。
- 结构复验：`stillReveal: true`、`compareVisible: true`、`reportVisible: true`、`failureVisible: true`；每一层均由玩家主动推进。
- 小屏复验：320×568 展开细节后“看完了”仍在当前视口；所有可见触控目标 ≥44 px，横向溢出为 0。
- 第五轮问题：原界面虽然层级已简化，但深蓝卡片和弹窗仍像普通现代游戏 UI，与用户提供的档案、飞行手册和人工批改参考不一致。
- 修复：开始页改为装订档案封面；主 HUD、人物框、口述记录与裁定栏转为纸张/登记表语汇；弹窗、时空图、揭晓与结算完整改为打孔档案页，并加入案卷编号、蓝墨批注、红色登记线和审批印章。
- 换肤复验：390×844 的主判断页仍以人物占据最大视觉面积；线索、状态与地图弹窗的装订孔不覆盖标题或按钮；320×568 的对白、线索计数和两个裁定按钮均在单屏内清晰可见。
